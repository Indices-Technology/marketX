# Weekly Sprint

> One sprint = one week. Every Monday we set the goal. Every Friday we review it.
> Sprint goal is set by the CEO. Tasks are assigned before work begins — no work starts without a card.

---

## Sprint Rhythm

| Day | Activity |
|---|---|
| Monday | Sprint planning — CEO sets goal, assigns tasks, confirms deadlines |
| Daily | Standup updates on Pumble |
| Friday | Sprint review — what shipped, what didn't, why |
| Friday | Next sprint goal drafted by CEO |

## Sprint Rules

1. Sprint goal is set by the CEO — it cannot be changed mid-sprint without CEO sign-off
2. A task not completed by Friday must have a written reason in the review
3. Three consecutive carry-overs of the same task triggers a performance or process review
4. No new features are added mid-sprint without removing something of equal weight
5. Engineers do not start work until the task card is in Notion with a deadline
